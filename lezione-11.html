L11 · Distribuzione geometrica

Quanti tentativi falliti devo affrontare prima del mio primo successo? La distribuzione geometrica è la risposta probabilistica a questa domanda: nasce da una sequenza di lanci «alla Bernoulli» (successo/fallimento indipendenti) e ha una proprietà sorprendente: «non ha memoria».

≈ 45–60 minuti 9 quiz 5 esercizi numpy, matplotlib, scipy.stats

1. Bernoulli, il punto di partenza

Un esperimento bernoulliano di parametro \(p \in [0, 1]\) è il più semplice esperimento aleatorio possibile: due soli esiti, chiamati convenzionalmente «successo» (probabilità \(p\)) e «fallimento» (probabilità \(1-p\)).

Lanciare una moneta equa equivale a un esperimento bernoulliano con \(p = 0.5\); estrarre un asso da un mazzo di 52 carte è un Bernoulli con \(p = 4/52 \approx 0.077\). 

import random as rnd

def bernoulli(p):
    """Restituisce 1 (successo) con probabilità p, altrimenti 0."""
    return 1 if rnd.random() < p else 0

rnd.seed(42)
prove = [bernoulli(0.3) for _ in range(20)]
print("20 prove con p=0.3:", prove)
print(f"Frequenza di successi: {sum(prove)/len(prove):.2f} (atteso {0.3:.2f})")

2. Definizione

Sia \(X\) la variabile aleatoria che conta il numero di insuccessi prima del primo successo in una sequenza di esperimenti bernoulliani indipendenti, ciascuno di parametro \(p\). Allora \(X \sim \mathrm{Geom}(p)\) ha supporto \(\{0, 1, 2, \dots\}\) e PMF

\[ f_X(i; p) = p (1 - p)^i \quad \text{per } i = 0, 1, 2, \dots \]

Il parametro deve essere \(p \in (0, 1]\): se \(p = 0\) il successo non arriva mai; se \(p = 1\) il primo esperimento è già un successo, quindi \(X\) vale sempre 0 (caso degenere).

Convenzione importante! Esistono due definizioni in letteratura (e nelle librerie):
  • Conta gli insuccessi (supporto \(\{0, 1, 2, \dots\}\)) — quella adottata qui e nel notebook;
  • Conta i tentativi totali, successo incluso (supporto \(\{1, 2, 3, \dots\}\)) — quella usata da scipy.stats.geom e in molti testi inglesi.
Differiscono per uno: \(X_{\text{tentativi}} = X_{\text{insuccessi}} + 1\). Cambiano anche valore atteso e varianza: presta attenzione alla convenzione!

Con la convenzione «numero di insuccessi prima del primo successo», qual è il supporto di \(X\)?

  • \(\{1, 2, 3, \dots\}\)
  • \(\{0, 1, 2, \dots\}\)
  • \(\{-1, 0, 1, 2, \dots\}\)
  • \([0, 1]\)

3. Simulare la geometrica

La definizione operativa suggerisce subito un algoritmo: continua a lanciare finché non esce successo, e conta i fallimenti incontrati strada facendo. 

import random as rnd

def geom(p):
    assert 0 < p <= 1, f"{p} non è un parametro valido per la geometrica."
    i = 0
    while rnd.random() >= p:
        i += 1
    return i

rnd.seed(0)
campioni = [geom(0.3) for _ in range(20)]
print("20 valori da Geom(0.3):", campioni)
print(f"\nMedia campionaria : {sum(campioni)/len(campioni):.3f}")
print(f"Valore atteso teorico (1-p)/p = {(1 - 0.3)/0.3:.3f}")

La libreria scipy.stats ha una versione molto più efficiente (e usa la convenzione «conta tentativi» partendo da 1). Per allinearci alla nostra definizione, basta sottrarre 1. 

from scipy.stats import geom as geom_sc
import numpy as np

p = 0.3
np.random.seed(7)
# scipy: supporto {1,2,...}; sottraiamo 1 per avere {0,1,...}
campioni = geom_sc(p).rvs(size=10000) - 1

print(f"Media campionaria : {campioni.mean():.4f}")
print(f"Atteso teorico    : {(1-p)/p:.4f}")
print(f"\nVarianza campionaria : {campioni.var():.4f}")
print(f"Atteso teorico       : {(1-p)/p**2:.4f}")

4. Funzione di massa di probabilità

La PMF \(f_X(i; p) = p(1-p)^i\) decresce in modo geometrico. Più \(p\) è grande, più la massa si concentra sui valori piccoli: il primo successo è probabile arrivi presto.

Verifichiamo che la somma totale faccia 1, sfruttando la serie geometrica \(\sum_{i=0}^{\infty} \alpha^i = \tfrac{1}{1-\alpha}\) per \(|\alpha| < 1\):

\[ \sum_{i=0}^{\infty} p (1-p)^i = p \cdot \frac{1}{1-(1-p)} = \frac{p}{p} = 1. \]

È proprio da qui che la distribuzione prende il nome.

import numpy as np
import matplotlib.pyplot as plt

def geom_pmf(i, p):
    return p * (1 - p) ** i if i >= 0 and i == int(i) else 0

ps = [0.2, 0.5, 0.8]
xs = np.arange(0, 12)

fig, axes = plt.subplots(1, 3, figsize=(11, 3.4), sharey=True)
for ax, p in zip(axes, ps):
    ys = [geom_pmf(int(x), p) for x in xs]
    ax.vlines(xs, 0, ys, color='steelblue')
    ax.plot(xs, ys, 'o', color='steelblue')
    ax.set_title(f"p = {p}")
    ax.set_xlabel("i")
    ax.set_ylim(0, 1)
axes[0].set_ylabel("P(X = i)")
plt.suptitle("PMF della Geometrica per diversi valori di p")
plt.tight_layout()
plt.show()
Verifica numerica. Sommare i primi termini della PMF deve dare un valore prossimo a 1 (e tendere a 1 al crescere di \(N\)). 
def geom_pmf(i, p):
    return p * (1 - p) ** i if i >= 0 and i == int(i) else 0

p = 0.3
for N in [5, 10, 20, 50, 100]:
    s = sum(geom_pmf(i, p) for i in range(N + 1))
    print(f"Somma dei primi {N+1:>3} termini: {s:.6f}")

Con \(p = 0.3\), qual è la probabilità di osservare il primo successo esattamente al terzo tentativo (cioè dopo 2 insuccessi)?

  • \(0.3\)
  • \(0.3 \cdot 0.7^{2} = 0.147\)
  • \(0.7^{3} = 0.343\)
  • \(0.3 \cdot 0.7^{3} = 0.103\)
\(X = 2\) (due insuccessi prima del successo). \(f(2; 0.3) = 0.3 \cdot 0.7^2 = 0.147\).

5. Valore atteso e varianza

Per la Geometrica «conta insuccessi» di parametro \(p\):

\[ \mathrm E(X) = \frac{1-p}{p}, \qquad \mathrm{Var}(X) = \frac{1-p}{p^2}. \]

Mostra la dimostrazione del valore atteso

Il punto chiave è il lemma: per \(\alpha \in (-1, 1)\), \(\sum_{i=0}^{\infty} i \alpha^i = \dfrac{\alpha}{(1-\alpha)^2}\). Si dimostra derivando la serie geometrica termine a termine. Poi:

\[ \mathrm E(X) = \sum_{i=0}^{\infty} i \cdot p (1-p)^i = p \cdot \frac{1-p}{(1-(1-p))^2} = p \cdot \frac{1-p}{p^2} = \frac{1-p}{p}. \]

import numpy as np
import matplotlib.pyplot as plt

p = np.arange(0.01, 1.01, 0.01)
EX  = (1 - p) / p
VarX = (1 - p) / p ** 2

fig, axes = plt.subplots(1, 2, figsize=(10, 3.6))
axes[0].plot(p, EX, color='steelblue')
axes[0].set_ylim(0, 50); axes[0].set_xlabel("p"); axes[0].set_ylabel("E(X)")
axes[0].set_title("Valore atteso (1-p)/p")

axes[1].plot(p, VarX, color='crimson')
axes[1].set_ylim(0, 50); axes[1].set_xlabel("p"); axes[1].set_ylabel("Var(X)")
axes[1].set_title("Varianza (1-p)/p²")

plt.tight_layout(); plt.show()
Interpretazione. Quando \(p\) è piccolo, il primo successo tarda molto: l'attesa cresce come \(1/p\) e la varianza come \(1/p^2\). Quando \(p \to 1\), entrambe vanno a zero: il primo successo arriva quasi sicuramente al primo colpo.

Con \(p = 0.1\), quanti insuccessi mi aspetto in media prima del primo successo?

  • 0.1
  • 1
  • 9
  • 10
\(\mathrm E(X) = (1-p)/p = 0.9/0.1 = 9\) insuccessi (il successo è al 10° tentativo).

Con \(p = 0.5\), valore atteso e deviazione standard sono…

  • \(\mathrm E = 1\), \(\sigma = 1\)
  • \(\mathrm E = 1\), \(\sigma = \sqrt{2} \approx 1.414\)
  • \(\mathrm E = 0.5\), \(\sigma = 0.5\)
  • \(\mathrm E = 2\), \(\sigma = 2\)
\(\mathrm E(X) = (1-p)/p = 1\); \(\mathrm{Var}(X) = (1-p)/p^2 = 2\); \(\sigma = \sqrt{2}\). 
import numpy as np
import matplotlib.pyplot as plt
import matplotlib.patches as patches

def gr_geom_pmf(p):
    fig, ax = plt.subplots(figsize=(7.5, 3.6))
    x = np.arange(0, 15)
    pmf = p * (1 - p) ** x

    avg   = (1 - p) / p
    stdev = ((1 - p) ** 0.5) / p
    ax.add_patch(patches.Rectangle(
        (avg - stdev, 0.95), 2 * stdev, 0.05, edgecolor='None', facecolor='green', alpha=0.6
    ))
    ax.plot([avg, avg], [0.9, 1.0], color='green')

    ax.vlines(x, 0, pmf, color='steelblue')
    ax.plot(x, pmf, 'o', color='steelblue')
    ax.set_ylim(0, 1); ax.set_xlim(-1, 16)
    ax.set_title(f"PMF di Geom({p}); banda verde = ±1σ attorno a E(X)")
    plt.tight_layout(); plt.show()

6. Funzione di ripartizione

La probabilità di superare \(n\) insuccessi è facile da calcolare:

\[ \mathrm P(X > n) = \sum_{i=n+1}^{\infty} p (1-p)^i = (1-p)^{n+1}. \]

Quindi la CDF, per \(n \in \mathbb N\), è

\[ F_X(n; p) = 1 - (1-p)^{n+1}. \]

Per un generico \(x \ge 0\), basta sostituire \(n = \lfloor x \rfloor\):

\[ F_X(x; p) = \big(1 - (1-p)^{\lfloor x \rfloor + 1}\big)\, \mathbb{1}_{x \ge 0}. \]

import numpy as np
import matplotlib.pyplot as plt

def geom_cdf(x, p):
    if x < 0:
        return 0
    return 1 - (1 - p) ** (int(x) + 1)

p = 0.5
xs = np.arange(-2, 11, 0.1)
ys = [geom_cdf(x, p) for x in xs]

plt.step(xs, ys, color='steelblue')
plt.title(f"CDF di Geom({p})")
plt.xlim(-2, 10); plt.ylim(-0.05, 1.05)
plt.xlabel("x"); plt.ylabel("F(x)")
plt.grid(alpha=0.3); plt.tight_layout(); plt.show()

Con \(p = 0.4\), \(\mathrm P(X \le 2)\) vale…

  • \(0.4\)
  • \(1 - 0.6^{2} = 0.64\)
  • \(1 - 0.6^{3} = 0.784\)
  • \(0.6^{3} = 0.216\)
\(F(2; p) = 1 - (1-p)^{2+1} = 1 - 0.6^3 = 0.784\). Equivale a dire che con probabilità ≈78% il primo successo arriva entro il terzo tentativo.

7. Assenza di memoria

La distribuzione geometrica è l'unica distribuzione discreta a possedere la proprietà di assenza di memoria:

\[ \mathrm P(X \ge x + y \mid X \ge x) = \mathrm P(X \ge y), \quad \forall x, y \ge 0. \]

In parole: dopo \(x\) insuccessi consecutivi, l'attesa ricomincia da zero — la moneta non «ricorda» quanto ha già perso e non è più «in debito» di successi. Il giocatore d'azzardo che pensa «sono in serie nera, ora deve per forza vincere» commette il gambler's fallacy.

Mostra la dimostrazione

Notiamo che \(\mathrm P(X \ge x) = (1-p)^{\lfloor x \rfloor}\) (la probabilità che i primi \(\lfloor x \rfloor\) tentativi siano tutti falliti). Allora

\[ \mathrm P(X \ge x+y \mid X \ge x) = \frac{(1-p)^{\lfloor x \rfloor + \lfloor y \rfloor}}{(1-p)^{\lfloor x \rfloor}} = (1-p)^{\lfloor y \rfloor} = \mathrm P(X \ge y). \]

import numpy as np

p = 0.3
np.random.seed(1)
# Simulo molte realizzazioni di Geom(p)
N = 200_000
campioni = np.random.geometric(p, size=N) - 1   # numpy.geometric usa supporto {1,2,...}

x, y = 5, 3
mask = campioni >= x
cond = (campioni[mask] >= x + y).mean()
no_cond = (campioni >= y).mean()

print(f"P(X >= {x+y} | X >= {x}) ≈ {cond:.4f}")
print(f"P(X >= {y})              ≈ {no_cond:.4f}")
print(f"Teoria: (1-p)^{y} = {(1-p)**y:.4f}")

Lancio una moneta truccata con \(p = 0.1\) di testa. Sono uscite 9 croci di seguito. Qual è la probabilità che la decima sia testa?

  • 1.00 — «deve» uscire testa
  • 0.90 — più alta del 0.1
  • 0.10 — non c'è memoria
  • Dipende dall'umore della moneta
Le prove sono indipendenti: la moneta non sa cosa è successo prima. Resta sempre 0.1.
Scope dell'assenza di memoria. Vale solo per la geometrica e la sua sorella continua, l'esponenziale. Quasi tutte le altre distribuzioni discrete (binomiale, Poisson, ipergeometrica, …) hanno memoria.

Esercizio 4 — Verifica empirica dell'assenza di memoria

Simula \(\mathrm{Geom}(0.2)\) e stima \(\mathrm P(X \ge 10 \mid X \ge 7)\). Confrontala con \(\mathrm P(X \ge 3)\) e con la formula teorica \((1-p)^3\). 

# Tua soluzione qui

import numpy as np

p = 0.2
N = 500_000
np.random.seed(42)
X = np.random.geometric(p, size=N) - 1

cond = (X[X >= 7] >= 10).mean()
marg = (X >= 3).mean()
teoria = (1 - p) ** 3

print(f"P(X >= 10 | X >= 7) ≈ {cond:.4f}")
print(f"P(X >= 3)            ≈ {marg:.4f}")
print(f"Teoria (1-p)^3      = {teoria:.4f}")

Esercizio 5 — Quanti tentativi per essere ragionevolmente sicuri?

Trova il più piccolo \(n\) tale che, con \(p = 0.05\), la probabilità di ottenere almeno un successo entro \(n\) tentativi sia almeno il 95%.

Suggerimento: \(\mathrm P(\text{almeno 1 successo in } n \text{ tentativi}) = 1 - (1-p)^n\). Risolvi rispetto a \(n\). 

# Tua soluzione qui

import math

p = 0.05
target = 0.95

# 1 - (1-p)^n >= target  ⇔  n >= log(1-target) / log(1-p)
n = math.ceil(math.log(1 - target) / math.log(1 - p))
print(f"n minimo = {n}")
print(f"Verifica: P(>=1 successo in {n}) = {1 - (1 - p)**n:.4f}")

Riepilogo

  • Definizione: \(X \sim \mathrm{Geom}(p)\) conta gli insuccessi prima del primo successo.
  • PMF: \(f_X(i; p) = p(1-p)^i\), \(i = 0, 1, 2, \dots\).
  • CDF: \(F_X(n; p) = 1 - (1-p)^{n+1}\).
  • Valore atteso: \(\mathrm E(X) = (1-p)/p\).
  • Varianza: \(\mathrm{Var}(X) = (1-p)/p^2\).
  • Assenza di memoria: la geometrica «dimentica» il passato.
  • Attenzione alle due convenzioni in giro per la letteratura: insuccessi vs tentativi.

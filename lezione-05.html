L05 · Indici di eterogeneità

Per i caratteri qualitativi nominali non ha senso parlare di varianza (manca la media). Servono nuovi indici che misurino quanto «sparpagliata» è una distribuzione di frequenze: Gini ed entropia. Vedremo poi come questi stessi indici sono il motore degli alberi di decisione, e infine introdurremo gli indici di concentrazione tipici dell'analisi della ricchezza.

≈ 50–65 minuti 9 quiz 4 esercizi Include scikit-learn

1. Perché un nuovo indice

Considera due dataset che descrivono il colore degli occhi di 10 supereroi:

  • A: tutti hanno gli occhi blu;
  • B: 2 blu, 2 verdi, 2 marroni, 2 grigi, 2 nocciola.

Sono entrambi qualitativi nominali, non possiamo calcolare una varianza. Ma intuitivamente B è molto più eterogeneo di A. Vogliamo un indice numerico che catturi questa idea: massima omogeneità (un solo valore osservato) ⇒ indice = 0; massima eterogeneità (tutti i valori equiprobabili) ⇒ indice massimo.

2. Indice di Gini

Date \(s\) modalità con frequenze relative \(f_1, \dots, f_s\), l'indice di eterogeneità di Gini è

\[ I = 1 - \sum_{i=1}^{s} f_i^2 \]

Valori notevoli:

  • massima omogeneità (una sola modalità con \(f_j = 1\)) ⇒ \(I = 0\);
  • massima eterogeneità (tutte le \(f_i = 1/s\)) ⇒ \(I = 1 - 1/s = (s-1)/s\).

Per ottenere un indice in [0, 1] indipendente dal numero di modalità, si normalizza:

\[ I' = \frac{s}{s-1} \cdot I \]

CSV = """\
Name;Eye color;Hair color;Publisher
Iron Man;Blue;Black;Marvel Comics
Hulk;Green;Green;Marvel Comics
Batman;Blue;Black;DC Comics
Superman;Blue;Black;DC Comics
Wonder Woman;Blue;Black;DC Comics
Spider-Man;Hazel;Brown;Marvel Comics
Wolverine;Blue;Black;Marvel Comics
Storm;Blue;White;Marvel Comics
Thor;Blue;Blond;Marvel Comics
Black Widow;Green;Auburn;Marvel Comics
Captain America;Blue;Blond;Marvel Comics
Joker;Green;Green;DC Comics
Aquaman;Blue;Blond;DC Comics
Catwoman;Green;Black;DC Comics
Flash;Green;Red;DC Comics
Cyborg;Brown;Black;DC Comics
Venom;Blue;Black;Marvel Comics
Doctor Strange;Grey;Black;Marvel Comics
Magneto;Blue;Grey;Marvel Comics
Penguin;Blue;Black;DC Comics
"""
import pandas as pd, io

heroes = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')

def gini(series):
    f = series.value_counts(normalize=True)
    return 1 - (f ** 2).sum()

def gini_norm(series):
    f = series.value_counts(normalize=True)
    s = len(f)
    return (1 - (f ** 2).sum()) * s / (s - 1) if s > 1 else 0.0

for col in ['Eye color', 'Hair color', 'Publisher']:
    print(f"{col:<12}  Gini = {gini(heroes[col]):.3f}   Gini norm = {gini_norm(heroes[col]):.3f}")

Vediamo il comportamento dell'indice nel caso più semplice di due sole modalità con frequenze \(f\) e \(1-f\): 

import numpy as np
import matplotlib.pyplot as plt

f = np.linspace(0, 1, 100)
gini_two = 1 - f**2 - (1 - f)**2

plt.plot(f, gini_two)
plt.title("Indice di Gini con 2 modalità: I = 1 - f² - (1-f)²")
plt.xlabel("f (frequenza della prima modalità)")
plt.ylabel("Indice di Gini")
plt.show()

Quanto vale l'indice di Gini per un campione in cui tutti gli individui hanno la stessa etichetta?

  • 1
  • 0
  • 0.5
  • Dipende dal numero di etichette possibili.
Una sola modalità con frequenza 1: \(I = 1 - 1^2 = 0\). Massima omogeneità ⇒ indice nullo.

Per s = 4 modalità tutte equifrequenti, quanto vale l'indice di Gini non normalizzato?

  • 1
  • 0.5
  • 0.75
  • 0.25
Con \(f_i = 1/4\) per ogni \(i\): \(I = 1 - 4 \cdot (1/4)^2 = 1 - 1/4 = 0.75\). Coincide con \((s-1)/s = 3/4\).

3. Entropia

L'entropia di una distribuzione discreta è

\[ H = -\sum_{i=1}^{s} f_i \log_2 f_i \]

(con la convenzione \(0 \log 0 = 0\)). Misurata in bit quando si usa il logaritmo in base 2. Anche qui: minimo \(0\) in caso di omogeneità, massimo \(\log_2 s\) in caso di equidistribuzione. La versione normalizzata è \(H' = H / \log_2 s\). 

CSV = """\
Name;Eye color;Publisher
Iron Man;Blue;Marvel Comics
Hulk;Green;Marvel Comics
Batman;Blue;DC Comics
Superman;Blue;DC Comics
Wonder Woman;Blue;DC Comics
Spider-Man;Hazel;Marvel Comics
Wolverine;Blue;Marvel Comics
Storm;Blue;Marvel Comics
Thor;Blue;Marvel Comics
Black Widow;Green;Marvel Comics
Captain America;Blue;Marvel Comics
Joker;Green;DC Comics
Aquaman;Blue;DC Comics
Catwoman;Green;DC Comics
Flash;Green;DC Comics
Cyborg;Brown;DC Comics
Venom;Blue;Marvel Comics
Doctor Strange;Grey;Marvel Comics
Magneto;Blue;Marvel Comics
Penguin;Blue;DC Comics
"""
import pandas as pd, io
import numpy as np

heroes = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')

def entropy(series, normalized=False):
    f = series.value_counts(normalize=True)
    h = -(f * np.log2(f)).sum()
    if normalized and len(f) > 1:
        h /= np.log2(len(f))
    return h

for col in ['Eye color', 'Publisher']:
    print(f"{col:<12}  H = {entropy(heroes[col]):.3f} bit"
          f"   H' = {entropy(heroes[col], normalized=True):.3f}")

Anche per l'entropia, nel caso di due modalità il grafico è "a campana": massimo in f = 1/2. 

import numpy as np
import matplotlib.pyplot as plt

def entropy_two(f):
    if f in (0, 1):
        return 0
    return -f * np.log2(f) - (1 - f) * np.log2(1 - f)

f = np.linspace(0.001, 0.999, 200)
H = [entropy_two(x) for x in f]

plt.plot(f, H)
plt.title("Entropia (2 modalità): massima in f=0.5")
plt.xlabel("f")
plt.ylabel("H (bit)")
plt.show()
Gini vs entropia. Le due funzioni hanno andamento qualitativo molto simile. In pratica producono risultati spesso indistinguibili, ma differiscono nelle code:l'entropia "punisce" più severamente i casi molto sbilanciati. Nei modelli di machine learning entrambe sono opzioni standard (parametro criterion in scikit-learn).

4. Alberi di decisione

Gli indici di eterogeneità sono il cuore degli alberi di decisione. L'algoritmo, in pratica, cerca quale attributo e quale soglia separano i dati in due gruppi più omogenei. Ripete poi la ricerca dentro ciascun gruppo, fino a raggiungere foglie sufficientemente pure.

Calcoliamo manualmente la "qualità" di una soglia sull'attributo Strength per separare buoni e cattivi: 

import pandas as pd
import matplotlib.pyplot as plt

DATA = pd.DataFrame({
    'Name':  ['Wonder Woman','Aquaman','Cyborg','Flash','Thor',
              'Black Manta','Penguin','Joker','Deathstroke','Bizarro'],
    'Strength': [100, 85, 80, 26, 100, 35, 9, 10, 55, 75],
    'label': ['good']*5 + ['bad']*5
}).set_index('Name')

def gini_two(p):
    return 1 - p**2 - (1 - p)**2

def split_score(thr):
    left  = DATA[DATA['Strength'] <= thr]
    right = DATA[DATA['Strength'] >  thr]
    g_left  = gini_two((left ['label'] == 'bad').mean()) if len(left)  else 0
    g_right = gini_two((right['label'] == 'bad').mean()) if len(right) else 0
    return (len(left) * g_left + len(right) * g_right) / len(DATA)

thresholds = range(5, 110, 5)
scores = [split_score(t) for t in thresholds]

plt.plot(list(thresholds), scores, marker='o')
plt.title("Gini medio dopo split su Strength ≤ thr")
plt.xlabel("threshold")
plt.ylabel("Gini medio (più basso = meglio)")
plt.show()

best = min(zip(scores, thresholds))
print(f"Migliore threshold: {best[1]} (Gini medio = {best[0]:.3f})")

Per costruire un albero "vero", non serve fare tutto a mano: scikit-learn mette a disposizione DecisionTreeClassifier. La cella sotto richiede di caricare il pacchetto scikit-learn (al primo run ci mette qualche secondo).

Black Manta;185;100;M;Brown;35;bad
Penguin;167;79;M;Blue;9;bad

Generalizzazione vs memorizzazione. Se valuti l'albero sugli stessi dati con cui l'hai costruito, il risultato è ottimisticamente alto: potrebbe avere semplicemente memorizzato il training set. Per una valutazione onesta serve un test set separato (lo vedremo in L08).

L'albero di decisione cerca lo split che:

  • massimizza il numero di foglie.
  • massimizza l'entropia dei figli.
  • minimizza l'eterogeneità media dei figli (Gini o entropia).
  • sceglie l'attributo a caso.
L'idea è ottenere figli il più possibile puri (omogenei rispetto all'etichetta). Si misura la purezza con Gini o entropia, e si sceglie lo split che la massimizza (= minimizza l'eterogeneità media pesata).

5. Indici di concentrazione (Lorenz)

Gli indici di concentrazione misurano quanto equamente una quantità (ricchezza, ricavi, ore di lavoro…) è distribuita su una popolazione. Sono concettualmente diversi dalla varianza: chiedono «pochi individui detengono quasi tutto?» piuttosto che «quanto si discostano dalla media?».

Ordinati i valori \(a_1 \le a_2 \le \dots \le a_n\), si definiscono:

  • \(F_i = i / n\): frequenza relativa cumulata fino all'\(i\)-esima osservazione;
  • \(Q_i = (\sum_{k=1}^i a_k) / \mathrm{TOT}\): quantità relativa cumulata.

La spezzata che congiunge i punti \((F_i, Q_i)\) è la curva di Lorenz. Quanto più si discosta dalla bisettrice (equidistribuzione), tanto più la distribuzione è concentrata. 

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

CSV = """\
Name;Strength
Iron Man;85
Hulk;100
Batman;26
Superman;100
Wonder Woman;100
Spider-Man;55
Wolverine;75
Storm;30
Thor;100
Black Widow;13
Captain America;19
Joker;10
Aquaman;85
Catwoman;14
Flash;26
Cyborg;80
Venom;85
Doctor Strange;10
Magneto;55
Penguin;9
Lex Luthor;10
Robin;14
"""
import io
strength = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')['Strength']

s_sorted = strength.sort_values().values
n = len(s_sorted)
F = np.arange(1, n+1) / n
Q = np.cumsum(s_sorted) / s_sorted.sum()

plt.fill_between(F, F, Q, alpha=0.2)
plt.plot([0, 1], [0, 1], '--', color='gray', label='equidistribuzione')
plt.plot(F, Q, color='blue', linewidth=2, label='curva di Lorenz')
plt.xlabel("F (frazione cumulata di individui)")
plt.ylabel("Q (frazione cumulata di forza)")
plt.legend()
plt.title("Curva di Lorenz: forza dei supereroi")
plt.show()

# Indice di concentrazione di Gini: G = 2 * sum(F - Q) / (n - 1)
G = 2 * (F[:-1] - Q[:-1]).sum() / (n - 1)
print(f"Indice di concentrazione di Gini: G = {G:.3f}  (0 = equa, 1 = massima)")
Due «Gini». Il coefficiente di concentrazione di Gini (questa sezione) è una cosa diversa dall'indice di eterogeneità di Gini visto sopra. Sono entrambi dovuti a Corrado Gini e si calcolano su frequenze cumulate, ma rispondono a domande diverse: «quanto è concentrata?» vs «quanto è varia?».

Una curva di Lorenz coincidente con la bisettrice indica:

  • massima concentrazione.
  • distribuzione perfettamente equa.
  • nessun dato.
  • solo una modalità.

6. Riepilogo & sfida finale

  • Per i dati qualitativi nominali non si calcola la varianza: si usano Gini ed entropia.
  • Gini: \(I = 1 - \sum f_i^2\). Versione normalizzata per confronti tra distribuzioni con \(s\) diversi.
  • Entropia: \(H = -\sum f_i \log_2 f_i\). Misurata in bit (logaritmo base 2).
  • Entrambi gli indici guidano gli split degli alberi di decisione (sklearn: criterion='gini' o 'entropy').
  • Per misurare concentrazione di una grandezza condivisibile: curva di Lorenz + coefficiente \(G\).

Hai un campione con una sola etichetta osservata. Quale coppia di valori è corretta?

  • Gini = 1, entropia = 1
  • Gini = 0, entropia = 0
  • Gini = 0.5, entropia = 0.5
  • Indeterminati

Sfida finale — Gini massimo

Costruisci una funzione che, dato un numero di modalità s, ritorni il valore massimo dell'indice di Gini (caso uniforme). Tracciane il grafico per s = 2, 3, …, 30. Cosa noti all'aumentare di s? 

import matplotlib.pyplot as plt

def gini_max(s):
    return (s - 1) / s

ss = list(range(2, 31))
ys = [gini_max(s) for s in ss]

plt.plot(ss, ys, marker='o')
plt.axhline(1, linestyle='--', color='gray')
plt.xlabel('s (numero di modalità)')
plt.ylabel('Gini massimo (s-1)/s')
plt.title('Gini massimo cresce verso 1 al crescere di s')
plt.show()

# All'aumentare di s il massimo si avvicina a 1 ma non lo raggiunge mai:
# ecco perché si normalizza per confrontare distribuzioni con s diversi.

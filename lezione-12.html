Tre nuovi modelli in L12.
  • Poisson(λ): numero di eventi rari in un intervallo; \(\mathrm E(X) = \lambda\), \(\mathrm{Var}(X) = \lambda\).
  • Esponenziale(λ): tempo (continuo) tra due eventi poissoniani consecutivi; assenza di memoria.
  • Teorema centrale del limite: la media campionaria standardizzata tende a una normale, qualunque sia la distribuzione di partenza.

Esercizio 1 · Poisson dalle basi

Sia \(X\) una variabile di Poisson di parametro \(\lambda\) (il numero medio di eventi che accadono in un intervallo prefissato).

1.1 Caratteristiche

  • Supporto: \(\mathbb N \cup \{0\} = \{0, 1, 2, \dots\}\).
  • PMF: \( \displaystyle p_X(k) = \frac{e^{-\lambda} \lambda^k}{k!} \), \(k \in \mathbb N\).
  • Valore atteso: \(\mathrm E(X) = \lambda\).
  • Varianza: \(\mathrm{Var}(X) = \lambda\).
Test rapido di Poisson. In una Poisson media e varianza coincidono: se sui dati osservati \(\mathrm{Var}(X) \approx \mathrm E(X)\), il modello è plausibile; se \(\mathrm{Var}(X) \gg \mathrm E(X)\) c'è overdispersion (es. binomiale negativa).

In una Poisson, qual è il rapporto \(\mathrm{Var}(X)/\mathrm E(X)\)?

  • Sempre < 1
  • Sempre = 1
  • Sempre > 1
  • Dipende da \(\lambda\)

1.2 Visualizziamo la PMF (\(\lambda = 5\))

import math, numpy as np
import matplotlib.pyplot as plt

def p_poisson(k, lam):
    return lam ** k * math.exp(-lam) / math.factorial(k)

lam = 5
xs = np.arange(0, 16)
ys = [p_poisson(int(k), lam) for k in xs]

plt.vlines(xs, 0, ys, color='steelblue')
plt.plot(xs, ys, 'o', color='steelblue')
plt.title(f"PMF di Poisson(λ={lam})")
plt.xlabel("k"); plt.ylabel("P(X = k)")
plt.tight_layout(); plt.show()

1.3 Stesso conto con scipy.stats

import scipy.stats as st

lam = 5
X = st.poisson(lam)

print(f"P(X = 5)   = {X.pmf(5):.4f}")
print(f"P(X > 6)   = {1 - X.cdf(6):.4f}")
print(f"x_0.8 (più piccolo k tale che F(k) ≥ 0.8) = {int(X.ppf(0.8))}")
print(f"verifica:  F(6) = {X.cdf(6):.4f},  F(7) = {X.cdf(7):.4f}")

Con \(\lambda = 5\), il valore \(x_{0.8}\) (più piccolo k tale che \(F_X(k) \ge 0.8\)) vale…

  • 5
  • 6
  • 7
  • 8
\(F(6) \approx 0.762 < 0.8\), \(F(7) \approx 0.867 \ge 0.8\). Quindi \(x_{0.8} = 7\).

1.4 Combinazione lineare \(Y = aX - b\)

Per linearità: \(\mathrm E(Y) = a \lambda - b\); \(\mathrm{Var}(Y) = a^2 \lambda\) (la traslazione \(b\) non cambia la varianza).

Se \(X \sim \mathrm{Poisson}(4)\) e \(Y = 3X - 2\), quanto vale \(\mathrm{Var}(Y)\)?

  • 4
  • 10
  • 34
  • 36
\(\mathrm{Var}(aX - b) = a^2 \mathrm{Var}(X) = 9 \cdot 4 = 36\).

Un'azienda gestisce alcune sorgenti di alta montagna. Un dispositivo filtra 5 litri d'acqua per volta e registra: NomeSorgente, Oro (numero di particelle d'oro in 5 L) e DurezzaAcqua. Vogliamo rispondere a una serie di domande puramente statistiche con questi dati.

Dataset embedded. Per girare in Pyodide sostituiamo il file ComposizioneAcqua.csv originale (1650 righe) con un campione sintetico ridotto ma costruito per riprodurre fedelmente le conclusioni qualitative del notebook.

2.1 Caricare e ispezionare il dataset

import pandas as pd, io
import numpy as np

# Costruiamo un dataset sintetico riproducibile
rng = np.random.default_rng(0)

# 5 sorgenti, ~50 osservazioni ciascuna (4 simili + sorgente5 diversa)
def make_block(name, n, lam_oro, mu_durezza, sigma_durezza):
    return pd.DataFrame({
        'NomeSorgente': [name] * n,
        'Oro':          rng.poisson(lam=lam_oro, size=n),
        'DurezzaAcqua': rng.normal(loc=mu_durezza, scale=sigma_durezza, size=n).round(2),
    })

acqua = pd.concat([
    make_block('Sorgente1', 50, 2.1, 14.0, 1.0),
    make_block('Sorgente2', 50, 2.0, 14.2, 0.9),
    make_block('Sorgente3', 50, 1.9, 13.9, 1.0),
    make_block('Sorgente4', 50, 2.0, 14.1, 0.95),
    make_block('Sorgente5', 50, 0.6, 8.5,  1.0),   # sorgente "diversa"
], ignore_index=True)

# Iniettiamo un po' di rumore sulla relazione oro/durezza, per coerenza
acqua['DurezzaAcqua'] = acqua['DurezzaAcqua'] - 0.4 * acqua['Oro']

print("\nPrime righe:")
print(acqua.head())

2.2 Sorgenti uniformemente rappresentate?

Per misurare quanto è uniforme la rappresentazione delle sorgenti calcoliamo l'indice di Gini normalizzato (vedi L05): vale 1 se le frequenze sono perfettamente uniformi e 0 se una sola modalità ha tutta la massa. 

import pandas as pd, io
import numpy as np

# Stesso dataset costruito sopra (ripetiamo per cella indipendente)
rng = np.random.default_rng(0)
def make_block(name, n, lam_oro, mu_durezza, sigma_durezza):
    return pd.DataFrame({
        'NomeSorgente': [name]*n,
        'Oro':          rng.poisson(lam=lam_oro, size=n),
        'DurezzaAcqua': rng.normal(loc=mu_durezza, scale=sigma_durezza, size=n).round(2),
    })
acqua = pd.concat([
    make_block('Sorgente1', 50, 2.1, 14.0, 1.0),
    make_block('Sorgente2', 50, 2.0, 14.2, 0.9),
    make_block('Sorgente3', 50, 1.9, 13.9, 1.0),
    make_block('Sorgente4', 50, 2.0, 14.1, 0.95),
    make_block('Sorgente5', 50, 0.6, 8.5,  1.0),
], ignore_index=True)

def gini(series):
    return 1 - sum(series.value_counts(normalize=True).map(lambda f: f**2))

def normalized_gini(series):
    s = series.nunique()
    return s * gini(series) / (s - 1)

print("Sorgenti distinte:", acqua['NomeSorgente'].nunique())
print(f"Gini normalizzato: {normalized_gini(acqua['NomeSorgente']):.4f}")
print("\nFrequenze per sorgente:")
print(acqua['NomeSorgente'].value_counts())

2.4 La distribuzione è compatibile con una Poisson?

Test rapido: se i dati seguono una Poisson, media e varianza sono uguali.

2.5 Stimatore: media campionaria, non distorto

Lo stimatore usato per \(\lambda\) è la media campionaria \( \overline X = \tfrac{1}{n} \sum_{i=1}^n X_i \). È non distorto perché \(\mathrm E(\overline X) = \mathrm E(X) = \lambda\). 

import pandas as pd, io
import numpy as np

rng = np.random.default_rng(0)
def make_block(name, n, lam_oro, mu_durezza, sigma_durezza):
    return pd.DataFrame({
        'NomeSorgente': [name]*n,
        'Oro':          rng.poisson(lam=lam_oro, size=n),
        'DurezzaAcqua': rng.normal(loc=mu_durezza, scale=sigma_durezza, size=n).round(2),
    })
acqua = pd.concat([
    make_block('Sorgente1', 50, 2.1, 14.0, 1.0),
    make_block('Sorgente2', 50, 2.0, 14.2, 0.9),
    make_block('Sorgente3', 50, 1.9, 13.9, 1.0),
    make_block('Sorgente4', 50, 2.0, 14.1, 0.95),
    make_block('Sorgente5', 50, 0.6, 8.5,  1.0),
], ignore_index=True)

n_missing = acqua['Oro'].isna().sum()
n         = len(acqua) - n_missing
lambda_hat = acqua['Oro'].mean()

print(f"Valori mancanti su 'Oro': {n_missing}")
print(f"Numerosità campione    : {n}")
print(f"Stima λ (= media camp.): {lambda_hat:.4f}")

La media campionaria è uno stimatore…

  • distorto, ma efficiente
  • non distorto del valore atteso \(\mathrm E(X)\)
  • distorto del valore atteso
  • non definito quando \(n < 30\)

3.5 Dalla Poisson all'esponenziale

Se il numero di particelle in 5 L è \(X \sim \mathrm{Poisson}(\lambda)\), il numero di litri tra una particella e la successiva è una variabile esponenziale di parametro \( \mu = \lambda / 5 \) (intensità per litro). La sua densità e CDF sono:

\[ f_T(t) = \mu e^{-\mu t}, \quad t \ge 0; \qquad F_T(t) = 1 - e^{-\mu t}. \]

Anche l'esponenziale ha la proprietà di assenza di memoria (versione continua): \(\mathrm P(T > s + t \mid T > s) = \mathrm P(T > t)\). 

import pandas as pd, io
import numpy as np
import scipy.stats as st

rng = np.random.default_rng(0)
def make_block(name, n, lam_oro, mu_durezza, sigma_durezza):
    return pd.DataFrame({
        'NomeSorgente': [name]*n,
        'Oro':          rng.poisson(lam=lam_oro, size=n),
        'DurezzaAcqua': rng.normal(loc=mu_durezza, scale=sigma_durezza, size=n).round(2),
    })
acqua = pd.concat([
    make_block('Sorgente1', 50, 2.1, 14.0, 1.0),
    make_block('Sorgente2', 50, 2.0, 14.2, 0.9),
    make_block('Sorgente3', 50, 1.9, 13.9, 1.0),
    make_block('Sorgente4', 50, 2.0, 14.1, 0.95),
    make_block('Sorgente5', 50, 0.6, 8.5,  1.0),
], ignore_index=True)

sorgente_5 = acqua[acqua['NomeSorgente'] == 'Sorgente5']
mu_5 = sorgente_5['Oro'].mean() / 5    # particelle per litro

print(f"μ stimato (particelle/L) per sorgente5: {mu_5:.4f}")
T = st.expon(scale=1/mu_5)             # scipy: scale = 1/μ

# Probabilità di non trovare oro nei prossimi 10 L
p10 = 1 - T.cdf(10)
print(f"\nP(T > 10 L) = {p10:.4f}")

# Assenza di memoria: P(T > 5+10 | T > 5) = P(T > 10)
p_cond = (1 - T.cdf(15)) / (1 - T.cdf(5))
print(f"P(T > 15 | T > 5) = {p_cond:.4f}  (uguale a P(T > 10), per assenza di memoria)")

Se eventi accadono come Poisson(λ) per intervallo di durata 5, l'intervallo tra eventi consecutivi è…

  • Geometrico, parametro λ
  • Esponenziale, parametro μ = λ/5
  • Normale di media 5/λ
  • Uniforme su [0, 5]

Quale di queste distribuzioni non ha l'assenza di memoria?

  • Geometrica
  • Esponenziale
  • Poisson
  • Sia geometrica che esponenziale ce l'hanno
La Poisson ha memoria: il numero di eventi in un intervallo dipende dall'ampiezza, non «si dimentica» del passato.

Esercizio 4 · TCL e profitto atteso

4.1 Errore di stima e teorema centrale del limite

Vogliamo sapere con quale probabilità la stima campionaria \(\overline X\) si discosta da \(\lambda\) per meno di \(\epsilon = 0.1\). Per il TCL, \( \overline X \) è approssimativamente normale di media \(\lambda\) e deviazione standard \(\sigma_X / \sqrt n\). Quindi:

\[ \mathrm P(|\overline X - \lambda| < \epsilon) \approx 2 \Phi\!\left(\frac{\epsilon \sqrt n}{\sigma_X}\right) - 1. \]

import pandas as pd, io
import numpy as np
import scipy.stats as st

rng = np.random.default_rng(0)

4.2 Quanti litri in più per arrivare al 90%?

Vogliamo \(\mathrm P(|\overline X - \lambda| < \epsilon) \ge 0.9\), ossia \(\Phi(\epsilon \sqrt n / \sigma_X) \ge 0.95\). Risolvendo per \(n\):

\[ n \ge \left( \frac{\sigma_X}{\epsilon} \, \Phi^{-1}(0.95) \right)^2. \]

import pandas as pd, io
import math
import numpy as np
import scipy.stats as st

rng = np.random.default_rng(0)
def make_block(name, n, lam_oro, mu_durezza, sigma_durezza):
    return pd.DataFrame({
        'NomeSorgente': [name]*n,
        'Oro':          rng.poisson(lam=lam_oro, size=n),
        'DurezzaAcqua': rng.normal(loc=mu_durezza, scale=sigma_durezza, size=n).round(2),
    })
acqua = pd.concat([
    make_block('Sorgente1', 50, 2.1, 14.0, 1.0),
    make_block('Sorgente2', 50, 2.0, 14.2, 0.9),
    make_block('Sorgente3', 50, 1.9, 13.9, 1.0),
    make_block('Sorgente4', 50, 2.0, 14.1, 0.95),
    make_block('Sorgente5', 50, 0.6, 8.5,  1.0),
], ignore_index=True)

sorgente_5 = acqua[acqua['NomeSorgente'] == 'Sorgente5']
sigma_x = sorgente_5['Oro'].std()
n_attuali = len(sorgente_5)
eps = 0.1

Z = st.norm()
n_0 = math.ceil((sigma_x / eps * Z.ppf(0.95))**2)

print(f"n minimo richiesto : {n_0}")
print(f"Già osservati      : {n_attuali}")
print(f"Mancanti           : {n_0 - n_attuali}")
print(f"Equivalenti a {(n_0 - n_attuali) * 5} L di acqua")

4.3 Profitto atteso e varianza

1000 L di acqua = 200 osservazioni da 5 L, ognuna \( \sim \mathrm{Poisson}(\lambda) \). Il numero totale di particelle d'oro è \( N = \sum_{i=1}^{200} X_i \sim \mathrm{Poisson}(200 \lambda) \). Se ogni particella vale 1.5 € e il dispositivo costa 1000 €:

\[ G = 1.5 N - 1000, \quad \mathrm E(G) = 1.5 \cdot 200\lambda - 1000 = 100(3\lambda - 10), \quad \mathrm{Var}(G) = 1.5^2 \cdot 200\lambda. \]

import pandas as pd, io
import numpy as np

rng = np.random.default_rng(0)
def make_block(name, n, lam_oro, mu_durezza, sigma_durezza):
    return pd.DataFrame({
        'NomeSorgente': [name]*n,
        'Oro':          rng.poisson(lam=lam_oro, size=n),
        'DurezzaAcqua': rng.normal(loc=mu_durezza, scale=sigma_durezza, size=n).round(2),
    })
acqua = pd.concat([
    make_block('Sorgente1', 50, 2.1, 14.0, 1.0),
    make_block('Sorgente2', 50, 2.0, 14.2, 0.9),
    make_block('Sorgente3', 50, 1.9, 13.9, 1.0),
    make_block('Sorgente4', 50, 2.0, 14.1, 0.95),
    make_block('Sorgente5', 50, 0.6, 8.5,  1.0),
], ignore_index=True)

lam_5 = acqua[acqua['NomeSorgente']=='Sorgente5']['Oro'].mean()

EG  = 100 * (3 * lam_5 - 10)
VarG = 1.5**2 * 200 * lam_5
print(f"λ stimato Sorgente5 : {lam_5:.4f}")
print(f"E(G)   = {EG:.2f} €")
print(f"Var(G) = {VarG:.2f}")
print(f"σ(G)   = {VarG**0.5:.2f} €")

if EG > 0:
    print("\n→ in media, attività potenzialmente in attivo (ma con grande incertezza).")
else:
    print("\n→ in media, attività in perdita: l'oro è troppo poco per ripagare il dispositivo.")

Per dimezzare l'errore standard di \(\overline X\), dobbiamo…

  • Raddoppiare \(n\)
  • Quadruplicare \(n\) (errore scala come \(1/\sqrt n\))
  • Raddoppiare \(\sigma\)
  • Niente: dipende dal modello

Sommando 200 variabili IID \(\mathrm{Poisson}(\lambda)\) si ottiene…

  • Una geometrica
  • Una normale (esattamente)
  • Una \(\mathrm{Poisson}(200\lambda)\)
  • Un'altra Poisson di parametro \(\lambda/200\)
La somma di Poisson indipendenti è ancora una Poisson, con parametro uguale alla somma dei parametri. Per il TCL, però, è anche approssimativamente normale.

Cose da saper fare alla fine del percorso:

  • Caricare un dataset, ispezionarlo, gestire i valori mancanti.

  • Contare con permutazioni, disposizioni, combinazioni.
  • Applicare il teorema di Bayes e usare un classificatore Naive Bayes.
  • Riconoscere la geometrica (numero di tentativi), la Poisson (numero di eventi rari) e l'esponenziale (tempo tra eventi).
  • Stimare un parametro con la media campionaria e capire perché è non distorta.
  • Usare il TCL per costruire intervalli di confidenza o calcolare la dimensione campionaria necessaria.

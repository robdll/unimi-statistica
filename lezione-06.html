L06 · Trasformazione dei dati

I valori così come arrivano non sono sempre adatti: troppo grandi, in scale diverse, distribuzioni asimmetriche. Le trasformazioni permettono di rimettere ordine senza cambiare la forma della distribuzione (le frequenze restano invariate, perché le funzioni che usiamo sono iniettive).

≈ 30–45 minuti 8 quiz 4 esercizi

1. Perché trasformare

Le ragioni più comuni:

  • Confronto: due variabili in unità diverse (cm vs kg) sono difficili da confrontare; portarle a una scala comune (es. [0, 1]) le rende paragonabili.
  • Forma: una distribuzione molto asimmetrica può diventare quasi simmetrica con una trasformazione logaritmica.
  • Leggibilità: i valori grandi e poco dispersi diventano più comodi se traslati o standardizzati.

In tutti i casi che vedremo qui, la trasformazione è iniettiva: osservazioni distinte restano distinte e le frequenze non cambiano. Cambieranno gli indici descrittivi (media, std, ecc.) e l'aspetto del grafico, non l'informazione contenuta.

2. Traslazione

Sostituiamo \(x \mapsto x' = x - k\) (o \(+k\)). Effetti:

  • media, mediana, quantili: traslano della stessa quantità;
  • varianza, std, IQR, range: invariati.
CSV = """\
Name;First appearance
Iron Man;1963
Hulk;1962
Spider-Man;1962
Wolverine;1974
Storm;1975
Thor;1962
Captain America;1941
Doctor Strange;1963
Magneto;1963
Batman;1939
Superman;1938
Wonder Woman;1941
Aquaman;1941
Flash;1956
Cyborg;1980
Joker;1940
Catwoman;1940
Robin;1940
Penguin;1941
"""
import pandas as pd, io
year = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')['First appearance']

# Traslo: parto da 0 con il minimo
year_t = year - year.min()

print("Originale:")
print(f"  media = {year.mean():.2f}, std = {year.std():.2f}, range = {year.max()-year.min()}")
print("Traslato (parte da 0):")
print(f"  media = {year_t.mean():.2f}, std = {year_t.std():.2f}, range = {year_t.max()-year_t.min()}")

Una traslazione di +10 modifica:

  • solo la varianza.
  • varianza e media.
  • solo media, mediana e quantili.
  • nulla.
Sommare una costante sposta la "centralità" di altrettanto, ma le distanze tra i punti restano uguali ⇒ varianza, std, IQR e range non cambiano.

3. Cambiamento di scala

Sostituiamo \(x \mapsto x' = x / h\) con \(h > 0\). Effetti:

  • media, mediana, quantili, IQR, range, std: scalati di \(1/h\);
  • varianza: scalata di \(1/h^2\) (ricorda: la varianza ha unità al quadrato).
CSV = """\
Name;Strength
Iron Man;85
Hulk;100
Spider-Man;55
Wolverine;75
Storm;30
Thor;100
Black Widow;13
Captain America;19
Doctor Strange;10
Magneto;55
Batman;26
Superman;100
Wonder Woman;100
Aquaman;85
Flash;26
Cyborg;80
Joker;10
Catwoman;14
Robin;14
Penguin;9
"""
import pandas as pd, io
s = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')['Strength']

s_norm01 = s / s.max()  # ora vive in [0, 1]
print("Originale:  ", s.head(3).values)
print("In [0, 1]: ", s_norm01.head(3).round(2).values)
print(f"\nstd originale: {s.std():.3f}")
print(f"std scalata : {s_norm01.std():.3f}  (= {s.std()/s.max():.3f})")
print(f"var originale: {s.var():.3f}")
print(f"var scalata : {s_norm01.var():.3f}  (= {s.var()/s.max()**2:.3f})")

Se divido tutti i dati per 10, la varianza:

  • resta uguale.
  • si divide per 10.
  • si divide per 100.
  • si moltiplica per 100.
La varianza è la media degli scarti al quadrato: scalando di \(1/h\), si scala di \(1/h^2\). Per \(h=10\), si divide per 100.

4. Cambiamento di origine e scala

Per portare un intervallo \([a, b]\) di valori in un nuovo intervallo \([c, d]\), si combina traslazione e scala:

\[ x' = c + \frac{d - c}{b - a}(x - a) \]

Casi particolari frequenti:

  • min-max scaling in [0, 1]: \(x' = (x - a) / (b - a)\);
  • scaling in [-1, 1]: \(x' = 2 (x - a) / (b - a) - 1\).
CSV = """\
Name;Strength
Iron Man;85
Hulk;100
Spider-Man;55
Wolverine;75
Storm;30
Thor;100
Doctor Strange;10
Penguin;9
Catwoman;14
Aquaman;85
"""
import pandas as pd, io
s = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')['Strength']

a, b = s.min(), s.max()
c, d = -10, 10
s_new = c + (d - c) / (b - a) * (s - a)

print(s_new.round(2))
print(f"\nrange originale: [{a}, {b}]   range nuovo: [{s_new.min():.1f}, {s_new.max():.1f}]")

5. Standardizzazione (z-score)

La standardizzazione è il caso particolare di trasformazione affine in cui usi media e deviazione standard:

\[ z = \frac{x - \overline x}{s_x} \]

Sui dati trasformati: media campionaria \(\approx 0\) e std \(\approx 1\). Utile per:

  • confrontare variabili in unità diverse;
  • fornire input "nettati" a molti algoritmi di machine learning (KNN, SVM, reti neurali…);
  • valutare quanto un'osservazione è «lontana» dalla media in unità di std.
CSV = """\
Name;Strength
Iron Man;85
Hulk;100
Spider-Man;55
Wolverine;75
Storm;30
Thor;100
Black Widow;13
Captain America;19
Doctor Strange;10
Magneto;55
Batman;26
Superman;100
Wonder Woman;100
Aquaman;85
Flash;26
Cyborg;80
Joker;10
Catwoman;14
Robin;14
Penguin;9
"""
import pandas as pd, io
s = pd.read_csv(io.StringIO(CSV), sep=';', index_col='Name')['Strength']
z = (s - s.mean()) / s.std()

print(z.round(2).head(10))
print()
print(f"media z = {z.mean():.2e}   (≈ 0, a meno di errori float)")
print(f"std   z = {z.std():.4f}    (≈ 1)")
Z-score utile per spotting outlier. Se \(|z| > 3\) per un dato, è plausibile classificarlo come "anomalo" (sotto l'ipotesi di distribuzione approssimativamente normale).

Dopo standardizzazione, la nuova varianza è:

  • 0
  • 1
  • la stessa di prima
  • indeterminata

Le z-score trasformano i dati in un'unità chiamata:

  • kg
  • cm
  • (adimensionale)
  • %
Il numeratore e il denominatore hanno la stessa unità di misura, quindi si semplificano. La z-score è adimensionale: «quante deviazioni standard di distanza dalla media».

6. Trasformazioni logaritmiche

Quando i dati spaziano su molti ordini di grandezza (popolazioni, redditi, prezzi…), \(x \mapsto \log x\) li "compatta" e spesso simmetrizza la distribuzione. La base del logaritmo (2, \(\mathrm e\), 10) è una scelta stilistica: cambia la scala dei valori trasformati ma non la forma della distribuzione. 

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# Distribuzione molto asimmetrica a destra (lognormale)
rng = np.random.default_rng(0)
x = pd.Series(rng.lognormal(mean=0, sigma=1, size=2000), name='x')

fig, axs = plt.subplots(1, 2, figsize=(9, 3.5))
x.hist(ax=axs[0], bins=40)
axs[0].set_title("x originale (skew destro)")

np.log10(x).hist(ax=axs[1], bins=40, color='#22c55e')
axs[1].set_title("log10(x): quasi simmetrica")
plt.tight_layout()
plt.show()

Le frequenze non cambiano (la trasformazione è iniettiva): è solo la scala dell'asse x che diventa più informativa.

Cautele. Il log è definito solo su valori positivi: se hai zero o negativi devi traslare prima (es. log(x + 1)) oppure considerare se quei valori hanno un senso.

7. Riepilogo & sfida finale

  • Le trasformazioni iniettive lasciano le frequenze invariate ma cambiano gli indici descrittivi.
  • Traslazione: cambia solo media/mediana/quantili (non var, std, IQR).
  • Scala (\(/h\)): divide media, std, IQR, range per \(h\); divide la varianza per \(h^2\).
  • Min-max: porta un intervallo \([a, b]\) in \([c, d]\); molto sensibile agli outlier.
  • Standardizzazione: \(z = (x - \bar x)/s_x\); media 0, std 1; più robusta del min-max.
  • Logaritmo: utile per dati con range esteso, comprime e simmetrizza.

Vuoi confrontare due caratteri in unità diverse (cm e kg) sullo stesso grafico. Cosa è più adatto?

  • Disegnarli così come sono.
  • Sottrarre il minimo a entrambi.
  • Standardizzare entrambi (z-score).
  • Trasformare in logaritmo.
